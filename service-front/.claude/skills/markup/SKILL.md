スクリーンショット画像をもとに、対象ファイルをマークアップ（HTML/CSS実装）します。

## 使い方

```
/markup <画像パス> <対象ファイル>
```

例:
- `/markup design.png src/components/Hero.tsx`
- `/markup screenshots/20240101/top.png src/pages/index.tsx`

## $ARGUMENTS

`$ARGUMENTS` の形式: `<画像パス> <対象ファイル>`

## 手順

### 1. 引数の解析

`$ARGUMENTS` を空白で分割し、最初の要素を画像パス、2番目を対象ファイルとして取得する。

引数が不足している場合:
- 「使い方: /markup <画像パス> <対象ファイル>」と出力して終了

### 2. 画像の確認

指定された画像ファイルが存在するか確認する。

```bash
ls <画像パス>
```

存在しない場合: 「スクリーンショットがありません: <画像パス>」と出力して終了。

### 3. 対象ファイルの確認

対象ファイルが存在するか確認する。存在しない場合はファイルの種類（拡張子）に応じて新規作成する。

### 4. 画像の分析

画像を読み込み、以下を把握する:
- レイアウト構造（ヘッダー・サイドバー・メインコンテンツ等）
- カラーパレット
- タイポグラフィ（フォントサイズ・太さ）
- コンポーネント境界
- スペーシング・マージン

### 5. マークアップ実装

対象ファイルの拡張子に応じて実装する:

| 拡張子 | 実装内容 |
|-------|---------|
| `.tsx` / `.jsx` | React コンポーネント + Tailwind CSS or CSS Modules |
| `.html` | セマンティックHTML + インラインスタイルまたは別途CSSファイル |
| `.vue` | Vue SFC形式 |
| `.css` / `.scss` | スタイルのみ |

既存ファイルがある場合は差分を最小に抑えて追記・修正する。

### 6. 完了報告

```
マークアップ完了
═══════════════════════════
画像: <画像パス>
対象: <対象ファイル>

実装内容:
- <コンポーネント・要素の一覧>

手動調整が必要な箇所:
- <フォントファミリー・アセット等>
```
