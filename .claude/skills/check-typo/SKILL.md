変更差分に含まれるタイポ・不要な文字変更をチェックします。

## 使い方

```
/check-typo
```

## 手順

### 1. 変更ファイルの取得

以下のコマンドで変更差分を取得する:

```bash
git diff HEAD                     # 全変更の差分詳細
git diff HEAD --name-only         # 変更ファイル一覧
```

変更ファイルが存在しない場合は「変更差分がありません」と出力して終了。

**除外対象**（以下のファイルはスキップ）:
- `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`
- `node_modules/**`, `dist/**`, `build/**`, `.next/**`
- バイナリファイル（画像、フォント等）

差分取得時に除外する:
```bash
git diff HEAD -- ':!package-lock.json' ':!yarn.lock' ':!pnpm-lock.yaml' ':!node_modules' ':!dist' ':!build' ':!.next'
```

### 2. タイポチェック

差分の追加行（`+` で始まる行）を対象に、以下のパターンを検出する。

**スペルミス（コード）:**
- 変数名・関数名のスペルミス（例: `fucntion` → `function`, `consoel` → `console`, `recieve` → `receive`）
- import文のパス誤り

**スペルミス（コメント・文字列）:**
- 日本語の誤字脱字
- 英単語のスペルミス

**全角/半角混在（コード内のみ）:**
- 全角スペース `　` の混入
- 全角英数字 `ａ-ｚ`, `０-９` の使用
- 全角記号 `（`, `）`, `，`, `；` の使用（コメント・文字列リテラル内は許可）

**引用符の誤用:**
- バッククォート `` ` `` とシングルクォート `'` の混在（テンプレートリテラル以外でのバッククォート使用）

**判定基準:**
- `.vscode/settings.json` の `cSpell.words` に登録されている単語はタイポとしない
- プロジェクト固有の略語・造語は警告レベルで報告する（Error にしない）

### 3. 不要な文字変更チェック

空白のみの変更を検出する:

```bash
git diff -w HEAD -- ':!package-lock.json' ':!yarn.lock' ':!pnpm-lock.yaml'
```

`git diff HEAD` の結果と `git diff -w HEAD` の結果を比較し、`-w` で差分が消える箇所を「空白のみの変更」として検出する。

**検出パターン:**
- 末尾の空白追加/削除のみ
- 改行コードの変更のみ（LF ↔ CRLF）
- インデントの不要な変更（タブ ↔ スペース）
- 空行の追加/削除のみ

**除外条件（意図的な変更として報告しない）:**
- `.prettierrc` / `.editorconfig` が存在し、そのルールに沿った変更
- リファクタリングに伴うインデント変更（コードブロックの移動を含む差分）

### 4. 出力フォーマット

```
タイポ・不要な文字変更チェック
═══════════════════════════════
対象: <ファイル数>ファイル

## タイポの可能性

### <ファイルパス>:<行番号>
<種別>: <誤> → <正>（または指摘内容）
   <該当行のコード>

（0件の場合は「タイポは検出されませんでした」と表示）

## 不要な文字変更

### <ファイルパス>
<種別>（<箇所数>箇所）
   - <行番号>行目: <内容>

（0件の場合は「不要な文字変更は検出されませんでした」と表示）

## サマリー
タイポ: <件数>件
不要な変更: <箇所数>箇所
合計: <合計>件の確認事項
```

### 5. 修正提案

問題が見つかった場合:
1. タイポは自動修正を提案
2. 不要な変更は削除を提案
