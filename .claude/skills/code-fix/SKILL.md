コード規約に基づいて現在のブランチのコードを修正します。

## 使い方

- `/code-fix` — 変更ファイル全体を対象に修正
- `/code-fix $ARGUMENTS` — 指定したファイルのみ修正（例: `src/components/Button.tsx`）

## 手順

### 1. 対象ファイルの特定

引数 `$ARGUMENTS` が指定されている場合:
- 指定ファイルのみを対象とする
- ファイルが存在しない場合は「指定ファイルが見つかりません: $ARGUMENTS」と出力して終了

引数がない場合:
```bash
git diff --name-only HEAD
```
で変更ファイル一覧を取得する。差分がなければ「修正対象のファイルがありません」と出力して終了。

### 2. コード規約の読み込み

対象ファイルの拡張子に応じて、該当する規約ファイルを読み込む:

| 拡張子 | 読み込む規約 |
|--------|------------|
| `.ts`, `.tsx` | `rules/typescript.md`, `rules/react.md` |
| `.css` | `rules/css.md` |
| `.html` | `rules/html.md` |
| `.php` | `rules/php.md` |

全ファイル共通で以下も読み込む:
- `.claude/CLAUDE.md` — プロジェクト共通規約
- `rules/accessibility.md` — アクセシビリティ規約（`.tsx`, `.html` の場合）
- `rules/readable-code.md` — 命名規則（全拡張子）

規約ファイルが一つも存在しない場合は「コード規約ファイルが見つかりません。`rules/` に規約を追加してください」と出力して終了。

**規約間の競合時**: `.claude/CLAUDE.md` > `rules/` 配下の個別規約。プロジェクト共通規約が最優先。

### 3. レビュー結果の参照

同一セッション内で直近に `/review` を実行した結果がある場合は、その Error・Warning 指摘を修正対象リストに追加する。`/review` を実行していない場合はこのステップをスキップする。

### 4. ファイルごとの修正

対象ファイルを1つずつ確認し、規約に違反している箇所を修正する。

**判定基準と修正ポリシー:**

| レベル | 判定基準 | 修正方法 |
|--------|---------|---------|
| **Error** | 型エラー、`any` の使用、未使用 import、セキュリティ脆弱性 | Edit ツールで自動修正する |
| **Warning** | 命名規則違反、宣言的スタイル違反（手続きループ→配列メソッド等）、早期リターン未使用 | Edit ツールで自動修正する。ただし変数・関数名の変更が他ファイルの import に波及する場合は Info に格下げし手動確認を促す |
| **Info** | コメントの改善提案、より良い命名の提案 | 修正せず、サマリーに「手動確認推奨」として記載 |

### 5. 修正内容のサマリー表示

```
コード修正完了
═══════════════════════════
対象: <ファイル数>ファイル
適用規約: <読み込んだ規約ファイル一覧>

修正内容:
- [ファイルパス:行番号] [修正内容]
...

手動確認が必要な箇所: <件数>件
- [ファイルパス:行番号] [Info内容]
```

## $ARGUMENTS

指定ファイルパス（省略可）。省略時は変更ファイル全体が対象。
