コード規約に基づいて現在のブランチのコードを修正します。

## 使い方

- `/code-fix` — 変更ファイル全体を対象に修正
- `/code-fix $ARGUMENTS` — 指定したファイルのみ修正（例: `src/components/Button.tsx`）

## 手順

### 1. 対象ファイルの特定

引数 `$ARGUMENTS` が指定されている場合:
- 指定ファイルのみを対象とする
- ファイルが存在しない場合は「指定ファイルが見つかりません: $ARGUMENTS」と出力して終了

引数がない場合:
```bash
git diff --name-only HEAD
```
で変更ファイル一覧を取得する。差分がなければ「修正対象のファイルがありません」と出力して終了。

### 2. コード規約の読み込み

以下のファイルを探索し、存在するものをすべて読み込む。

| ファイル | 内容 |
|---------|------|
| `rules/react.md` | React コーディング規約 |
| `rules/typescript.md` | TypeScript コーディング規約 |
| `rules/html.md` | HTML コーディング規約 |
| `rules/css.md` | CSS コーディング規約 |
| `rules/php.md` | PHP コーディング規約 |
| `.claude/CLAUDE.md` | プロジェクト共通規約 |

規約ファイルが一つも存在しない場合は「コード規約ファイルが見つかりません。`rules/` に規約を追加してください」と出力して終了。

### 3. レビュー結果の参照

会話内に `/review` の結果がある場合は、その指摘事項も修正対象に含める。

### 4. ファイルごとの修正

対象ファイルを1つずつ確認し、規約に違反している箇所を修正する。

修正の優先度:
1. **Error** — 必ず修正（バグ・型エラー・規約違反）
2. **Warning** — 可能な限り修正（命名規則・パターン違反）
3. **Info** — 任意（スタイル改善）

### 5. 修正内容のサマリー表示

```
コード修正完了
═══════════════════════════
対象: <ファイル数>ファイル
適用規約: <読み込んだ規約ファイル一覧>

修正内容:
- [ファイルパス:行番号] [修正内容]
...

修正完了 / 手動確認が必要な箇所: <件数>
```


## $ARGUMENTS

指定ファイルパス（省略可）。省略時は変更ファイル全体が対象。
