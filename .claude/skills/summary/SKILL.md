現在のブランチの変更内容をもとに、PRのディスクリプションをマークダウン形式で生成します。

## 使い方

```
/summary
```

## 手順

### 1. 変更内容の収集

```bash
git diff main...HEAD --stat       # 変更ファイル一覧
git diff main...HEAD              # 差分詳細
git log main..HEAD --oneline      # コミット一覧
git branch --show-current         # 現在のブランチ名
```

差分がない場合は「PR対象の変更がありません」と出力して終了。

### 2. 影響URLの特定

`/check:diff-impact` スキルの手順に従い、変更ファイルから影響を受けるURLを特定する。

特定できた場合はPRディスクリプションの「動作確認URL」セクションに以下の形式で追加する:

```markdown
## 動作確認URL

対象ページ：トップページ（https://example.com/）
対象ページ：ブログ一覧（https://example.com/blog/）
```

ページタイトルは `<title>` タグや見出しから取得する。ローカル開発サーバーのURLは `package.json` の `scripts` から自動検出し、見つからない場合は `http://localhost:3000` をデフォルトとする。

URLが特定できない場合（ページファイルやコンポーネントの変更がない場合）はこのセクションを省略する。

### 3. PR テンプレートの確認

`.github/PULL_REQUEST_TEMPLATE.md` が存在するか確認する。

**テンプレートが存在する場合:**
- ファイルを読み込み、その構造・見出し・チェックリストをそのまま使用する
- 各セクションを変更内容に基づいて埋める
- テンプレートの構造は一切変えない

**テンプレートが存在しない場合:**
- 以下のデフォルトフォーマットを使用する

### 4. PRディスクリプションの生成

#### テンプレートなし（デフォルト）

以下に「動作確認URL」セクションを含む:

ブランチ名のプレフィックスを参考にタイプを判定:

| プレフィックス | タイプ |
|--------------|--------|
| `feature/` | 新機能 |
| `fix/` | バグ修正 |
| `refactor/` | リファクタリング |
| `chore/` | 雑務・設定変更 |
| `docs/` | ドキュメント |
| `update/` | バージョンアップ |

```markdown
## 概要

<変更の目的・背景を1〜3文で説明>

## 変更内容

- <変更点1>
- <変更点2>
- <変更点3>

## 変更ファイル

| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| `src/...` | 追加/修正/削除 | <概要> |

## 動作確認URL

<!-- 影響URLが特定できた場合のみ記載。特定できない場合はこのセクションを削除 -->

対象ページ：<ページタイトル>（https://localhost:3000/example）

## 動作確認

- [ ] ローカルで動作確認済み
- [ ] 既存機能への影響なし
- [ ] テスト追加/更新済み（該当する場合）

## 備考

<レビュアーへの補足・注意事項（あれば）>
```

### 5. 出力

生成したマークダウンをそのままコンソールに出力する（コピペしてPRに貼り付け可能な形式）。

```
PRディスクリプション生成完了
コピーしてGitHubのPR説明欄に貼り付けてください。
═══════════════════════════
<マークダウン本文>
```

### 6. ファイル保存

生成したマークダウンをプロジェクトルートの `.claude/summary/` ディレクトリに保存する。

**ファイル名の形式:** `YYYYMMDD_<内容>.md`

- `YYYYMMDD`: 今日の日付（例: `20260413`）
- `<内容>`: ブランチ名または変更内容を表す短い英数字スラッグ（スペース・スラッシュはハイフンに置換、最大30文字）

**例:**
- `20260413_feature-user-auth.md`
- `20260413_fix-login-redirect.md`

**手順:**
1. `.claude/summary/` ディレクトリが存在しない場合は作成する
2. ファイル名を決定して、生成したマークダウン本文をそのまま書き込む
3. 保存完了後に以下を出力する:

```
保存先: .claude/summary/<ファイル名>
```
