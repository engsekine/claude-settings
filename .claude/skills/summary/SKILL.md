現在のブランチの変更内容をもとに、PRのディスクリプションをマークダウン形式で生成します。

## 使い方

```
/summary
```

## 手順

### 1. 変更内容の収集

```bash
git diff main...HEAD --stat       # 変更ファイル一覧
git diff main...HEAD              # 差分詳細
git log main..HEAD --oneline      # コミット一覧
git branch --show-current         # 現在のブランチ名
```

差分がない場合は「PR対象の変更がありません」と出力して終了。

**出力**: ブランチ名、変更ファイル一覧、差分詳細、コミット一覧 → 手順2〜4の入力

### 2. 影響URLの特定

`/check:diff-impact` スキルの SKILL.md を読み、その手順2「影響URLの特定」のロジックに従って影響URLを特定する（スキルを呼び出すのではなく、同じロジックを実行する）。

影響URLが1件以上ある場合は「動作確認URL」セクションに追加する。0件の場合はセクションごと省略する。

### 3. PR テンプレートの確認

`.github/PULL_REQUEST_TEMPLATE.md` が存在するか確認する。

**テンプレートが存在する場合:**
- テンプレートの見出し構造・チェックリストをそのまま維持する
- 各セクションの内容を変更差分に基づいて埋める
- テンプレートに存在しないセクションは追加しない
- テンプレートのセクションで埋める内容がない場合は「該当なし」と記載する

**テンプレートが存在しない場合:**
- 手順4のデフォルトフォーマットを使用する

### 4. PRディスクリプションの生成

#### タイプ判定

ブランチ名のプレフィックスでタイプを判定する:

| プレフィックス | タイプ |
|--------------|--------|
| `feature/` | 新機能 |
| `fix/` | バグ修正 |
| `refactor/` | リファクタリング |
| `chore/` | 雑務・設定変更 |
| `docs/` | ドキュメント |
| `update/` | バージョンアップ |

**プレフィックスがない場合**: コミットメッセージの最頻プレフィックス（`feat:`, `fix:` 等）からタイプを推定する。それでも判定できない場合は「変更」とする。

#### デフォルトフォーマット（テンプレートなしの場合）

```markdown
## 概要

<変更の目的・背景を1〜3文で説明>

## 変更内容

- <変更点1>
- <変更点2>
- <変更点3>

## 変更ファイル

| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| `src/...` | 追加/修正/削除 | <概要> |

## 動作確認URL

対象ページ：<ページタイトル>（http://localhost:3000/<パス>）

## 動作確認

- [ ] ローカルで動作確認済み
- [ ] 既存機能への影響なし
- [ ] テスト追加/更新済み（該当する場合）

## 備考

<レビュアーへの補足・注意事項（あれば）>
```

### 5. 出力

生成したマークダウンをコンソールに出力する:

```
PRディスクリプション生成完了
コピーしてGitHubのPR説明欄に貼り付けてください。
═══════════════════════════
<マークダウン本文>
```

### 6. ファイル保存

生成したマークダウンを `.claude/summary/` に保存する。

**ファイル名**: `YYYYMMDD_<スラッグ>.md`
- `YYYYMMDD`: 実行日（例: `20260420`）
- `<スラッグ>`: ブランチ名からプレフィックス（`feature/` 等）を除き、`/` をハイフンに置換。最大30文字

**例**: `20260420_user-auth.md`

ディレクトリが存在しない場合は作成する。同名ファイルが存在する場合は末尾に `-2`, `-3` を付与する。
