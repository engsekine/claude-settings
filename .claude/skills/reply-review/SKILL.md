PRのレビューコメントに対して、コミット差分・変更ログを根拠に納得感のある返信ドラフトを生成する。

## 入力パターン

- `/reply-review <レビューコメント本文を貼り付け>` — 貼り付けたコメントに対する返信ドラフトを生成

レビューコメントはユーザーがGitHubからコピーしてそのまま貼り付ける。PR番号やAPIからの取得は行わない。

## 手順

### 1. 変更コンテキストの収集

現在のブランチのコミット差分・ログから変更の全体像を把握する。

```bash
git diff main...HEAD --stat
git diff main...HEAD
git log main..HEAD --oneline
git diff --cached --stat    # ステージング済みの変更があれば併せて確認
git diff --cached
```

差分がない場合は「変更差分がありません。対象ブランチで実行してください。」と伝えて終了。

**出力**: 変更ファイル一覧、差分詳細、コミット一覧 → 手順2〜3の入力

### 2. レビューコメントの分析

貼り付けられたレビューコメントを読み解き、**指摘のポイント**を箇条書きで整理する。

- レビュアーが何を問題視・疑問視しているか
- 具体的にどの設計判断・コードに対する指摘か
- 暗黙の前提や期待があれば明示する

**レビュアー名の抽出**: コメント内に `@username` や「○○さん」があればそれを使用。不明な場合は「レビュアー」とする。

**複数コメントの区切り**: 以下のいずれかで判断する:
- 空行2行以上で区切られている
- `---` や `===` などの水平線
- 明らかに異なるファイル・トピックへの言及

### 3. PRの変更範囲との関係を分析する

手順1の差分で、指摘対象のファイル・コードが含まれているかを確認する。

| 分類 | 判定方法 | 説明 |
|------|---------|------|
| **変更範囲内** | 指摘対象のファイルパスが `git diff main...HEAD --name-only` に含まれる | このブランチで追加・変更したコードへの指摘 |
| **変更範囲外** | 指摘対象のファイルパスが差分に含まれない | 既存コードへの指摘 |
| **関連あり** | ファイルは差分に含まれないが、変更したファイルが import/依存している | 今回の変更と関連する既存コードへの指摘 |

### 4. 返信ドラフトの生成

以下の判定基準で返信パターンを選択する:

| パターン | 判定基準 | 返信の方向性 |
|---------|---------|-------------|
| **変更範囲外** | 手順3で「変更範囲外」と判定 | 同意しつつスコープを区切る。別issue対応を提案 |
| **質問への回答** | 指摘ではなく疑問・確認の形式 | 端的に回答し、必要なら補足する |
| **指摘が正しい** | 指摘内容がコード規約・ベストプラクティスに沿っており、明確に修正すべき | 認めて修正方針を示す |
| **議論が必要** | 指摘に一理あるが、複数の妥当なアプローチがありトレードオフがある | 選択肢とトレードオフを提示して相談する |
| **意図的な実装** | コミットメッセージや差分から明確な意図が読み取れ、技術的理由がある | コミット・差分を引用して理由を説明する |

**判定順序**: 上から順に評価し、最初に該当したパターンを採用する。「変更範囲外」は他パターンより優先して判定する。

**トーンのルール:**
- です・ます調
- 根拠を示す（「〜だと思います」より「〜のため」）
- 防御的にならない（NG例: 「〜ですが、そもそも〜」「〜は仕方なく〜」。OK例: 「ご指摘の通りです」「〜のため〜としました」）
- 具体的な改善案があれば添える

### 5. 出力フォーマット

```
これは<レビュアー名>からのレビューコメントですね。<指摘対象>について<質問/指摘>されています。

レビュアーの指摘のポイント:

1. <ポイント1>
2. <ポイント2>

補足: <変更範囲内/変更範囲外/関連ありの判定結果と、その根拠>

レビュアーに対しては、例えば以下のように返答できます:

> <返信ドラフト本文>

このコメントへの対応方針について、何か相談はありますか？
```

**複数コメントが貼り付けられた場合:**
- コメントごとに上記の構成で出力する
- コメント間は `---` で区切る

## $ARGUMENTS

- `/reply-review <レビューコメント>` — 貼り付けたレビューコメントに対する返信ドラフトを生成
