PRのレビューコメントに対して、コミット差分・変更ログを根拠に納得感のある返信ドラフトを生成する。

## 入力パターン

- `/reply-review <レビューコメント本文を貼り付け>` — 貼り付けたコメントに対する返信ドラフトを生成

レビューコメントはユーザーがGitHubからコピーしてそのまま貼り付ける。PR番号やAPIからの取得は行わない。

## 手順

### 1. 変更コンテキストの収集

現在のブランチのコミット差分・ログから変更の全体像を把握する。

```bash
# ブランチの差分
git diff main...HEAD --stat
git diff main...HEAD

# コミットログ
git log main..HEAD --oneline

# ステージング済みの変更があれば併せて確認
git diff --cached --stat
git diff --cached
```

差分がない場合は「変更差分がありません。対象ブランチで実行してください。」と伝えて終了。

### 2. レビューコメントの分析

貼り付けられたレビューコメントを読み解き、**指摘のポイント**を箇条書きで整理する。

- レビュアーが何を問題視・疑問視しているか
- 具体的にどの設計判断・コードに対する指摘か
- 暗黙の前提や期待があれば明示する

### 3. PRの変更範囲との関係を分析する

手順1で収集したコミット差分を確認し、指摘対象が以下のどれに該当するか判定する。

| 分類           | 説明                                                           |
| -------------- | -------------------------------------------------------------- |
| **変更範囲内** | このブランチで追加・変更したコードに対する指摘                 |
| **変更範囲外** | このブランチでは変更していない既存コードに対する指摘           |
| **関連あり**   | 直接は変更していないが、今回の変更と関連する既存コードへの指摘 |

指摘対象のファイルやコードが差分に含まれているかを具体的に確認する。変更範囲外・関連ありの場合は、実際に変更した箇所との違いを補足として明記する。

### 4. 返信ドラフトの生成

以下の方針で返信文を生成する。

**返信パターンの判定:**

1. **指摘が正しい場合** — 素直に認め、修正方針を示す
   - 「ご指摘ありがとうございます。{具体的な修正方針}で対応します。」

2. **意図的な実装の場合** — 理由を根拠とともに説明する
   - コミットメッセージや差分から意図を引用
   - コードの前後関係から技術的な理由を説明

3. **変更範囲外の改善提案** — 同意しつつスコープを区切る
   - 「ご指摘の通り{問題点}です。{改善の方向性}が適切だと思います。別issueで対応を検討します。」

4. **議論が必要な場合** — トレードオフを提示して相談する
   - 「{案A}と{案B}のトレードオフがあり、{理由}で現在の実装を選択しましたが、{代替案}も検討できます。」

5. **質問への回答** — 端的に回答し、必要なら補足する

**返信のトーン:**

- 丁寧だが簡潔（冗長な敬語は避ける）
- 根拠を示す（「〜だと思います」より「〜のため」）
- 防御的にならない（指摘を歓迎する姿勢）
- 技術的に具体的な改善案があれば添える（`loading="lazy"` や `Intersection Observer` など具体名を出す）

### 5. 出力フォーマット

以下の構成で自然な文章として出力する。**セクション見出しや装飾記号は使わず、会話的なトーンで書く。**

```
これは@{レビュワー名}さんからのレビューコメントですね。{指摘対象}について{質問/指摘}されています。

レビュアーの指摘のポイント:

1. {ポイント1}
2. {ポイント2}
3. {ポイント3}

補足: {PRの変更範囲との関係。変更範囲外であればその旨を明記し、実際に変更した箇所との違いを説明する}

レビュアーに対しては、例えば以下のように返答できます:

> {返信ドラフト本文。根拠を含め、具体的な改善策があればそれも盛り込む}

このコメントへの対応方針について、何か相談はありますか？
```

レビュワー名が不明な場合は「レビュアー」とする。

**複数コメントが貼り付けられた場合:**

- コメントごとに上記の構成で出力する
- コメント間は水平線（`---`）で区切る

## $ARGUMENTS

- `/reply-review <レビューコメント>` — 貼り付けたレビューコメントに対する返信ドラフトを生成
