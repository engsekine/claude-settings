# コードレビュースキル

ブランチの変更内容をサブエージェントで並列チェックし、人間のレビューを支援するレポートを生成する。

## 入力パターン

- `/review` - 現在のブランチのmainとの差分をレビュー
- `/review TASK-XXX` - 指定タスクに関連する変更をレビュー（タスクのスコープ・完了条件も参照）

## 前提確認

- mainブランチとの差分またはステージング済み変更が存在すること（どちらもなければ「レビュー対象がありません」と伝えて終了）

## 手順

### 1. 変更内容の収集（内部処理）

以下を実行し、変更の全体像を把握する。

- `git diff --cached --stat` でステージング済みの変更ファイル一覧を取得
- `git diff --cached` でステージング済みの差分の詳細を取得
- `git diff main...HEAD --stat` でコミット済みの変更ファイル一覧を取得
- `git diff main...HEAD` でコミット済みの差分の詳細を取得
- 上記を統合してレビュー対象の全差分とする
- `git log main..HEAD --oneline` でコミット一覧を取得
- タスクIDが指定されている場合、`docs/tasks/TASK-XXX.md` を読む

### 2. レビュールールの収集（内部処理）

以下のファイルを探索し、存在するものをすべてレビュー基準として読み込む。

| ファイル                          | 内容                               |
| --------------------------------- | ---------------------------------- |
| `.coderabbit.yaml`                | CodeRabbitのレビュールール         |
| `CLAUDE.md`                       | プロジェクトのコードスタイル・規約 |
| `eslint.config.*` / `.eslintrc.*` | ESLint設定（ルームルートのみ）     |
| `tsconfig.json`                   | TypeScript設定                     |
| `docs/designs/`                   | 該当する設計書                     |
| `docs/specs/`                     | 該当する仕様書                     |

### 3. カスタムサブエージェントによる並列レビュー

以下の3つのカスタムサブエージェント（`.claude/agents/` に定義済み）を `Agent` ツールで**並列に**実行する。

**① code-reviewer**（コード品質チェック）

```
「以下の変更ファイルをレビューしてください。
 変更ファイル: [変更ファイル一覧]」
```

→ エージェント側にチェック観点・規約参照ルールが定義済み

**② doc-checker**（設計書・仕様書との整合性チェック）

```
「以下の変更ファイルと設計書・仕様書を比較してください。
 変更ファイル: [変更ファイル一覧]
 設計書: [該当する設計書パス]
 仕様書: [該当する仕様書パス]」
```

→ エージェント側にチェック観点・報告フォーマットが定義済み

**③ impact-analyzer**（変更影響の分析）

```
「以下の変更が他のファイルや機能に与える影響を分析してください。
 変更ファイル: [変更ファイル一覧]」
```

→ エージェント側に分析観点・報告フォーマットが定義済み

### 4. レビューレポートの統合

3つのサブエージェントの結果を統合し、以下のフォーマットで報告する。

```
📋 コードレビューレポート
═══════════════════════════

🔍 対象: main...<ブランチ名> (<コミット数>コミット, <変更ファイル数>ファイル)（ステージング済みを含む）
📎 タスク: TASK-XXX（該当する場合）

─────────────────────────
🔴 Error（修正必須）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
🟡 Warning（修正推奨）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
🔵 Info（参考情報）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
📄 ドキュメント整合性
─────────────────────────
- 設計書: [整合 or 差異あり（内容）]
- 仕様書: [整合 or 差異あり（内容）]

─────────────────────────
💥 変更影響
─────────────────────────
- [影響を受けるファイル・機能]（or 影響なし）

─────────────────────────
✅ 総合判定: [LGTM / 要修正 / 要確認]
─────────────────────────
```

### 5. 問題がある場合のアクション提案

Errorがある場合:

```
修正するには:
  /fix [問題の説明]
```

ドキュメント差異がある場合:

```
ドキュメントを更新する必要があります。
修正後に /commit してください。
```

## $ARGUMENTS

- `/review` - 現在のブランチの変更をレビュー
- `/review TASK-XXX` - 指定タスクのスコープに沿ってレビュー
