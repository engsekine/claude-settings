# コーディングスキル

コーディングタスクを実行する際の総合的なガイドラインとベストプラクティス。

## サブガイド

詳細なガイドは `guides/` ディレクトリを参照:

| ガイド                | 説明                                 |
| --------------------- | ------------------------------------ |
| `guides/fix.md`       | コード修正の手順とベストプラクティス |
| `guides/implement.md` | 新機能実装の手順とベストプラクティス |

## 基本原則

### 1. 作業前の確認

- 現状を把握する（既存コードを読む）
- 計画を立ててから実装する
- テストを書いてから実装コードを変更する

### 2. コーディング規約

- `rules/coding/` 配下の規約に準拠する
- TypeScript strict mode を使用
- async/await を優先（Promise チェーンは避ける）
- 関数型スタイル優先（副作用は明示的に）

### 3. コミットメッセージ

- `feat:` 新機能
- `fix:` バグ修正
- `docs:` ドキュメントのみ
- `refactor:` リファクタリング（機能変更なし）
- `test:` テストの追加・修正
- `chore:` ビルド設定・依存パッケージ更新

## 入力パターン

- `〜を実装して` → `guides/implement.md` を参照
- `〜を修正して` → `guides/fix.md` を参照
