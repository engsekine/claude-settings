指定した2ブランチ間の差分をレビューします。

## 使い方

- `/review` — 現在のブランチと main の差分をレビュー
- `/review $ARGUMENTS` — `<ベースブランチ> <比較ブランチ>` を指定してレビュー（例: `/review main feature/xxx`）

## $ARGUMENTS

`<ベースブランチ> <比較ブランチ>`（省略時は `main` と現在のブランチ）

## 手順

### 1. ブランチの解決

引数がある場合: `$ARGUMENTS` を空白で分割し、1番目をベース、2番目を比較ブランチとする。
引数がない場合: ベース = `main`、比較 = 現在のブランチ。

```bash
git branch --show-current  # 現在のブランチ確認
```

差分がない場合は「レビュー対象がありません（差分なし）」と出力して終了。

### 2. 変更内容の収集

```bash
git diff <ベース>...<比較> --stat   # 変更ファイル一覧
git diff <ベース>...<比較>          # 差分詳細
git log <ベース>..<比較> --oneline  # コミット一覧
```

### 3. レビュールールの収集

以下のファイルを探索し、存在するものをすべてレビュー基準として読み込む。

| ファイル | 内容 |
|---------|------|
| `rules/coding/react.md` | React 規約 |
| `rules/coding/typescript.md` | TypeScript 規約 |
| `rules/coding/html.md` | HTML 規約 |
| `rules/coding/css.md` | CSS 規約 |
| `rules/coding/php.md` | PHP 規約 |
| `.claude/CLAUDE.md` | プロジェクト共通規約 |
| `eslint.config.*` / `.eslintrc.*` | ESLint 設定 |
| `tsconfig.json` | TypeScript 設定 |

### 4. レビューレポートの生成

```

═══════════════════════════

🔍 対象: <ベース>...<比較> (<コミット数>コミット, <変更ファイル数>ファイル)

─────────────────────────
🔴 Error（修正必須）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
🟡 Warning（修正推奨）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
🔵 Info（参考情報）
─────────────────────────
- [ファイル:行番号] [説明]

─────────────────────────
✅ 総合判定: [LGTM / 要修正 / 要確認]
─────────────────────────
```

### 5. 問題がある場合のアクション提案

Error がある場合:
```
修正するには: /code-fix <ファイルパス>
```
