現在のブランチの変更内容をもとに、PRのディスクリプションをマークダウン形式で生成します。

## 使い方

```
/summary
```

## 手順

### 1. 変更内容の収集

```bash
git diff main...HEAD --stat       # 変更ファイル一覧
git diff main...HEAD              # 差分詳細
git log main..HEAD --oneline      # コミット一覧
git branch --show-current         # 現在のブランチ名
```

差分がない場合は「PR対象の変更がありません」と出力して終了。

### 2. PR テンプレートの確認

`.github/PULL_REQUEST_TEMPLATE.md` が存在するか確認する。

**テンプレートが存在する場合:**
- ファイルを読み込み、その構造・見出し・チェックリストをそのまま使用する
- 各セクションを変更内容に基づいて埋める
- テンプレートの構造は一切変えない

**テンプレートが存在しない場合:**
- 以下のデフォルトフォーマットを使用する

### 3. PRディスクリプションの生成

#### テンプレートなし（デフォルト）

ブランチ名のプレフィックスを参考にタイプを判定:

| プレフィックス | タイプ |
|--------------|--------|
| `feature/` | 新機能 |
| `fix/` | バグ修正 |
| `refactor/` | リファクタリング |
| `chore/` | 雑務・設定変更 |
| `docs/` | ドキュメント |
| `update/` | バージョンアップ |

```markdown
## 概要

<変更の目的・背景を1〜3文で説明>

## 変更内容

- <変更点1>
- <変更点2>
- <変更点3>

## 変更ファイル

| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| `src/...` | 追加/修正/削除 | <概要> |

## 動作確認

- [ ] ローカルで動作確認済み
- [ ] 既存機能への影響なし
- [ ] テスト追加/更新済み（該当する場合）

## 備考

<レビュアーへの補足・注意事項（あれば）>
```

---

### 4. 出力

生成したマークダウンをそのままコンソールに出力する（コピペしてPRに貼り付け可能な形式）。

```
📝 PRディスクリプション生成完了
コピーしてGitHubのPR説明欄に貼り付けてください。
═══════════════════════════
<マークダウン本文>
```
