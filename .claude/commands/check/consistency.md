変更差分に含まれる表記ゆれをチェックします。

## 使い方

```
/check:consistency
```

## 手順

### 1. 変更ファイルの取得

```bash
git diff --name-only HEAD
```

変更ファイルが存在しない場合は「変更差分がありません」と出力して終了。

### 2. 差分内容の取得

```bash
git diff HEAD
```

### 3. 表記ゆれチェック

差分内容から以下のパターンをチェック:

#### よくある表記ゆれパターン

**日本語:**
- ユーザー vs ユーザ
- サーバー vs サーバ
- エラー vs エラ
- データベース vs DB
- アプリケーション vs アプリ
- コンポーネント vs component

**英語:**
- color vs colour
- center vs centre
- initialize vs initialise

**記号:**
- 〜 vs ～ (波ダッシュ vs 全角チルダ)
- ー vs − (長音符 vs マイナス記号)
- ・ vs · (中黒 vs 中点)

**単位:**
- 10px vs 10 px
- 100% vs 100 %

#### チェック方法

1. 変更差分から追加行（`+` で始まる行）を抽出
2. よくある表記ゆれパターンに一致する文字列を検出
3. プロジェクト全体で同じ単語の異なる表記を検索

```bash
# 例: "ユーザー" と "ユーザ" の両方が使われているかチェック
grep -rn "ユーザー" . --include="*.tsx" --include="*.ts" --include="*.md"
grep -rn "ユーザ[^ー]" . --include="*.tsx" --include="*.ts" --include="*.md"
```

### 4. プロジェクト固有の用語チェック

`.vscode/settings.json` の `cSpell.words` を参照し、プロジェクトで使用される用語を確認。

### 5. 出力フォーマット

```
🔍 表記ゆれチェック
═══════════════════════════════

## 変更ファイル
変更ファイル数: 5

## 表記ゆれ検出結果

### ユーザー vs ユーザ
- src/pages/user.tsx:10 → ユーザー (追加)
- src/components/UserList.tsx:5 → ユーザ (既存)
- src/hooks/useUser.ts:23 → ユーザー (既存)

推奨: **ユーザー** に統一
理由: プロジェクト内で3箇所中2箇所が「ユーザー」を使用

### データベース vs DB
- src/config/database.ts:12 → データベース (追加)
- src/utils/db.ts:5 → DB (既存)

推奨: **データベース** に統一
理由: 正式名称の使用を推奨

## チェック結果サマリー

ℹ️  表記ゆれ: 2件
📝 修正推奨: 3箇所

合計: 2件の表記ゆれ
```

### 6. 修正提案

表記ゆれが見つかった場合:
1. プロジェクト内での使用頻度を集計
2. 多数派または正式名称への統一を提案
3. 一括置換コマンドを提示（実行は確認後）

```bash
# 例: "ユーザ" → "ユーザー" への一括置換
find . -name "*.tsx" -o -name "*.ts" | xargs sed -i '' 's/ユーザ\([^ー]\)/ユーザー\1/g'
```

## 除外設定

以下のファイルはチェック対象外:
- `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`
- `node_modules/**`
- `dist/**`, `build/**`, `.next/**`
- バイナリファイル (画像、フォントなど)

## 注意事項

- プロジェクト固有の用語は `.vscode/settings.json` の `cSpell.words` を参照
- 固有名詞（商品名、サービス名など）は除外
- 変数名・関数名のキャメルケース・スネークケースは除外
- 一括置換は提案のみ、実行は確認後に行う
