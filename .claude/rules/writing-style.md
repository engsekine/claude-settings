# 文書作成スタイルガイド

## ファイル形式

### Markdown ファイル

- **文字エンコーディング:** UTF-8（BOM なし）を使用する
- **改行コード:** LF を使用する
- **ファイル末尾:** 必ず改行を1つ入れる

### エンコーディング確認方法

```bash
# ファイルのエンコーディングを確認
file <ファイル名>

# 期待される出力
# UTF-8 text（正しい）
# data（文字化けの可能性）
```

### 文字化け時の対処法

文字化けしたファイルは削除して再作成する

## Markdown 記法

### 見出し

- `#` の後にスペースを1つ入れる
- 見出しレベルは飛ばさない

### リスト

- 箇条書きは `-` を使用
- インデントはスペース2つ

### コードブロック

- 言語指定を必ず行う

## 日本語表記

### 句読点

- 句点: 。
- 読点: 、

### 数字・英字

- 原則として半角を使用する

## 注意事項

- **常にUTF-8エンコーディングを使用する**
- 機種依存文字は使用しない
