# ドキュメント記述スタイルガイド

Claude Code プロジェクトにおけるドキュメント（スキル定義、ガイド、コマンド等）の記述スタイル規約。

## 基本原則

### 文字エンコーディング

- **UTF-8** を使用する
- BOM なし
- 改行コードは LF（`\n`）

### 絵文字の使用禁止

**一部環境で文字化けが発生するため、絵文字は使用しない。**

```markdown
[NG] 悪い例:
❌ エラー
✅ 成功
🔍 検索中

[OK] 良い例:
[NG] エラー
[OK] 成功
検索中...
```

### 代替表現

絵文字の代わりに以下の表現を使用:

| 絵文字 | 代替表現 |
|--------|----------|
| ✅ | `[OK]` / `成功` / `完了` |
| ❌ | `[NG]` / `エラー` / `失敗` |
| ⚠️ | `[WARNING]` / `警告` / `注意` |
| 🔍 | `検索中` / `確認中` |
| 📝 | `メモ` / `注記` |
| 🔴 | `[ERROR]` / `重要` |
| 🟡 | `[WARN]` / `注意` |
| 🟢 | `[INFO]` / `情報` |
| 📋 | `リスト` / `計画` |

## マークダウン記法

### 見出し

```markdown
# H1: ページタイトル（1ファイルに1つのみ）
## H2: メインセクション
### H3: サブセクション
#### H4: 詳細セクション
```

### コードブロック

言語指定を必ず含める:

```markdown
​```typescript
const example = "code";
​```

​```bash
git status
​```
```

### リスト

- 箇条書きは `-` を使用（`*` や `+` は使わない）
- ネストは4スペースでインデント

```markdown
- 項目1
    - サブ項目1-1
    - サブ項目1-2
- 項目2
```

### 強調

- **太字**: `**テキスト**`
- *斜体*: `*テキスト*`（日本語では使用頻度低）
- `コード`: バッククォート

### リンク

```markdown
[リンクテキスト](URL)
[ファイル参照](.claude/guides/frontend.md)
```

## 文体とトーン

### 基本文体

- **常体（だ・である調）** を使用
- 「です・ます調」は使わない

```markdown
[NG] この機能は便利です。
[OK] この機能は便利である。
[OK] この機能は便利。
```

### 命令形

手順やガイドでは命令形を使用:

```markdown
[OK] ファイルを作成する
[OK] 次のコマンドを実行する
[OK] 以下を確認:
```

### 簡潔さ

- 冗長な表現を避ける
- 一文を短く保つ
- 箇条書きを活用する

## ファイル構造

### スキル定義ファイル

```markdown
---
name: skill-name
description: スキルの説明（絵文字なし）
---

# スキル名

スキルの概要説明。

## 入力パターン

- パターン1
- パターン2

## 手順

### 1. ステップ1

説明...

### 2. ステップ2

説明...
```

### ガイドファイル

```markdown
# ガイドタイトル

概要説明。

## セクション1

内容...

### サブセクション

詳細...

## 参考

- 関連リンク
- 補足情報
```

## コメントと注釈

### コードコメント内の説明

```markdown
​```typescript
// [注意] この処理は副作用がある
function example() {
  // ...
}
​```
```

### 補足情報

```markdown
> **注意:** 重要な補足事項
>
> 複数行にわたる場合は、このように記述する。
```

## 特殊文字

### 使用可能な記号

以下の記号は使用可能:

- 区切り線: `─────────`（罫線素片）
- 強調枠: `═══════════`（二重線）
- チェックボックス: `- [ ]` / `- [x]`
- 矢印: `→` / `←`
- 引用符: `「」` / `『』`

### 使用禁止の記号

- 絵文字全般（❌ ✅ ⚠️ 等）
- 装飾的な特殊文字（✨ 🎉 等）
- 環境依存文字

## テーブル

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |
```

- ヘッダー行は必須
- 区切り線は `|---|` 形式
- セル内のテキストは左寄せ（デフォルト）

## ファイルパス表記

```markdown
`/path/to/file.ts`
`.claude/skills/fix/SKILL.md`
```

- バッククォートで囲む
- 絶対パスまたはプロジェクトルートからの相対パス

## 禁止事項

1. **絵文字の使用**: 文字化けの原因となる
2. **HTML タグの直接記述**: マークダウン記法を優先
3. **全角スペース**: 半角スペースまたはタブを使用
4. **不要な空行**: 連続する空行は1行まで

## チェックリスト

新しいドキュメントを作成する際の確認項目:

- [ ] UTF-8 エンコーディングで保存されているか
- [ ] 絵文字を使用していないか
- [ ] コードブロックに言語指定があるか
- [ ] 常体（だ・である調）で統一されているか
- [ ] ファイルパスがバッククォートで囲まれているか
- [ ] 見出しレベルが適切か（H1は1つのみ）
- [ ] 文字化けする特殊文字を使用していないか
