# コーディングスキル集

修正履歴から学んだ繰り返し発生しやすいパターンを記録します。

## 修正パターン

### 命名規則

- **問題**: ディレクトリ名やファイル名のタイポ（例: `cording` vs `coding`）
- **解決**: 正しいスペルに統一する
- **理由**: パス参照の整合性を保つため

### パス参照

- **問題**: 存在しないファイルやディレクトリを参照している
- **解決**: 参照先の存在を確認し、必要に応じてファイルを作成する
- **理由**: ドキュメントの信頼性を保つため

### コミットメッセージ

- **問題**: 一貫性のないコミットメッセージ形式
- **解決**: Conventional Commits形式に統一する（feat, fix, docs, refactor, test, chore）
- **理由**: コミット履歴の可読性を向上させるため

## 追加ルール

新しいパターンが見つかった場合、以下の形式で追記してください:

### [カテゴリ名]

- **問題**: [問題の説明]
- **解決**: [解決方法]
- **理由**: [なぜそうするのか]
