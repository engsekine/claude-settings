# コーディングスキル集

修正指示から学んだパターンを蓄積し、コーディング品質の向上に活用します。
`/code-fix` コマンド実行時に自動的に追記されます。

---

## コード修正時の影響チェックフロー

コードを修正する際は、必ず以下の手順で影響範囲を確認してから修正を行う。

### 1. 修正前の影響範囲調査

#### 関数/クラスの使用箇所を検索
```bash
# Grepで全ての呼び出し元を検索
grep -r "関数名\|クラス名" --include="*.ts" --include="*.tsx"
```

#### 型定義の参照を確認
- TypeScriptの型が他のファイルで使われているか
- エクスポートされている場合は特に注意

#### 依存関係の確認
- 修正対象が依存している他のコード
- 修正対象に依存している他のコード

### 2. 影響度の評価

**🔴 高リスク (慎重に修正)**
- 多数のファイルで使用されている
- 公開API・共通コンポーネント
- グローバルな型定義

**🟡 中リスク (影響箇所を確認)**
- 複数ファイルで使用されている
- 内部APIだが使用箇所が多い

**🟢 低リスク (通常の修正)**
- 限定的な使用箇所
- 内部実装のみ
- プライベート関数

### 3. 修正実行

1. メイン修正を実施
2. 影響を受ける箇所の連鎖修正
3. 型エラー・参照エラーのチェック

### 4. 修正後の影響確認

**差分の全体的な影響をチェック:**

```bash
# 変更されたファイルを確認
git diff --name-only

# 各変更ファイルについて影響URLを確認
/check:diff-impact
```

**チェック項目:**
- Next.js App Router: `app/**/page.tsx` → URL影響
- Next.js Pages Router: `pages/**/*.tsx` → URL影響
- コンポーネント: 使用箇所のページへの影響
- グローバルファイル: 全ページへの影響
  - `app/layout.tsx`, `pages/_app.tsx`
  - `globals.css`, `tailwind.config.*`

### 5. テスト推奨箇所の提示

修正完了時に以下を提示:
- 影響を受けるURL一覧
- テストすべき機能
- 確認が必要なビルドエラー
