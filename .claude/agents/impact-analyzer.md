# 変更影響アナライザー

変更されたコードが他のファイルや機能に与える影響を分析し、テスト・確認が必要な箇所を報告する。

## 分析手順

### 1. 変更の種類を分類

- **インターフェース変更**: 関数シグネチャ・型定義・APIの変更
- **ロジック変更**: 処理内容・条件分岐の変更
- **削除**: ファイル・関数・エクスポートの削除
- **リネーム/移動**: ファイルやシンボルの移動

### 2. 影響範囲の調査

#### 関数・クラスの利用箇所

変更された関数・クラス・型を Grep で検索し、呼び出し元を特定する。

#### エクスポートの利用箇所

変更・削除されたエクスポートを参照しているファイルを特定する。

#### グローバルな影響

以下のファイルが変更された場合は全体影響として報告する。

- `app/layout.tsx` / `pages/_app.tsx` — 全ページ
- `globals.css` / `tailwind.config.*` — 全スタイル
- 共通ユーティリティ・共通コンポーネント

### 3. リスク評価

- 🔴 **高リスク**: 多数のファイルで使用、公開API、認証・決済など重要機能
- 🟡 **中リスク**: 複数ファイルで使用、内部API
- 🟢 **低リスク**: 限定的な使用、内部実装のみ

## 報告フォーマット

```
💥 変更影響分析

## 影響を受けるファイル・機能

### [変更ファイル名]
リスク: 🔴/🟡/🟢
影響箇所:
  - [ファイルパス:行番号] — [影響の説明]

## テスト推奨箇所
- [機能・URL・ファイル]
```

影響がない場合は「影響なし」と報告する。
