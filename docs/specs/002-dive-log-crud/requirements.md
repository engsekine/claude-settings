# 002 ログ CRUD - 要件

## 概要

ログインしたユーザーが自分のダイビングログを作成・閲覧・編集・削除できる。

## ユーザーストーリー

- ダイバーとして、潜ったダイブの詳細を記録したい
- ダイバーとして、過去のログを一覧で振り返りたい
- ダイバーとして、過去のログを編集・削除したい
- ダイバーとして、日付やポイント名でログを検索したい

## 受け入れ条件

### 一覧（/dives）

- 認証済みユーザーが `/dives` を開いたとき、システムは自分のログを日付降順で 20 件ずつ表示する
- 21 件以上あるとき、システムは「もっと見る」ボタンで次ページを取得する
- ログが 0 件のとき、システムは「最初のログを記録しよう」CTA を表示する
- 検索ボックスに日付範囲・ポイント名を入力したとき、システムは絞り込んだ結果を表示する

### 詳細（/dives/[id]）

- 自分の dive_id で詳細を開いたとき、システムは全項目を表示する
- 他人の dive_id で詳細を開いたとき、システムは 404 を返す（RLS による）
- 詳細画面に「編集」「削除」ボタンを表示する

### 新規作成（/dives/new）

- 必須項目を入力して送信したとき、システムは dive を保存し `/dives/[id]` にリダイレクトする
- 必須項目が欠けているとき、システムは該当フィールドにエラーメッセージを表示する
- `max_depth_m` が 0 以下のとき、システムはエラーを表示する
- `bottom_time_min` が 0 以下のとき、システムはエラーを表示する

### 編集（/dives/[id]/edit）

- 自分の dive_id で編集を保存したとき、システムは更新し `/dives/[id]` にリダイレクトする
- 他人の dive_id で編集を試みたとき、システムは 404 を返す（RLS による）

### 削除

- 詳細画面で削除ボタンを押し確認ダイアログで OK したとき、システムは dive を削除し `/dives` にリダイレクトする
- キャンセルしたとき、システムは何もしない

## 必須項目

- `dive_date`（潜水日）
- `location`（エリア / ポイント名）
- `max_depth_m`（最大水深、> 0）
- `bottom_time_min`（潜水時間、≧ 1）

それ以外は任意。

## 対象外（Phase 1）

- 写真アップロード
- スポットマスタとの紐付け
- PDF 出力
- 公開機能
- ログのエクスポート / インポート
- バディとの相互記録
