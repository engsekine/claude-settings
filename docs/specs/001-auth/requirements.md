# 001 認証 - 要件

## 概要

メール + パスワードでユーザーがアプリを利用できるようにする。Supabase Auth を利用する。

## ユーザーストーリー

- 新規ユーザーとして、アカウントを作成してログを残したい
- 既存ユーザーとして、ログインして自分のログにアクセスしたい
- ログインしたユーザーとして、安全にログアウトしたい
- パスワードを忘れたユーザーとして、メール経由でリセットしたい

## 受け入れ条件

### サインアップ

- ユーザーが `/signup` で姓・名（漢字）、姓・名（ローマ字）、ニックネーム、生年月日、性別、メール、パスワードを入力したとき、システムは Supabase にユーザーを作成し確認メールを送信する。同時に `public.user_details` に姓名・ニックネーム・生年月日・性別を保存する
- 姓または名が未入力または 50 文字を超えるとき、システムはエラーメッセージを表示しサインアップを拒否する
- 姓（ローマ字）または名（ローマ字）が半角英字以外を含む、または 50 文字を超えるとき、システムはエラーメッセージを表示しサインアップを拒否する
- ニックネームが未入力または 50 文字を超えるとき、システムはエラーメッセージを表示しサインアップを拒否する
- 生年月日が未入力、未来日付、または 1900-01-01 より前のとき、システムはエラーメッセージを表示しサインアップを拒否する
- 性別は男性 / 女性 / 未回答 の 3 択から選択する。未選択を許容せずデフォルトは未回答とする
- パスワードが 6 文字未満のとき、システムはエラーメッセージを表示しサインアップを拒否する
- 既に登録済みのメールアドレスのとき、システムは "このメールアドレスは既に登録されています" を表示する
- サインアップ要求成功時、システムは「確認メールを送信しました」と入力されたメールアドレスを表示する（この時点ではログイン状態にしない）
- ユーザーが確認メール内のリンクをクリックしたとき、システムは `/api/auth/callback` でセッションを発行し `/dives` にリダイレクトする
- 確認リンクが無効・期限切れのとき、システムは `/login?error=auth_callback_failed` にリダイレクトする

### ログイン

- ユーザーが `/login` で正しい認証情報を入力したとき、システムはセッションを作成し `/dives` にリダイレクトする
- 認証情報が誤っているとき、システムは "メールアドレスまたはパスワードが間違っています" を表示する
- 連続失敗時、Supabase のレート制限に従い一時的にロックされる

### ログアウト

- ログインユーザーがログアウトボタンを押したとき、システムはセッションを破棄し `/login` にリダイレクトする

### パスワードリセット

- ユーザーが `/reset-password` でメールアドレスを入力したとき、システムはリセットリンクを含むメールを送信する
- 登録されていないメールアドレスでも、情報漏洩防止のため成功メッセージを表示する
- リセットリンクから新パスワードを設定したとき、システムは更新し `/login` にリダイレクトする

### 認証ガード

- 未認証ユーザーが `/dives` 配下にアクセスしたとき、システムは `/login` にリダイレクトする
- 認証済みユーザーが `/login` `/signup` にアクセスしたとき、システムは `/dives` にリダイレクトする

## 対象外（Phase 1）

- ソーシャルログイン
- MFA / 2FA
- メールアドレス変更
- アカウント削除
- 招待機能
